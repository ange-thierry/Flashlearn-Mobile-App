# Flashcard Learning App

A Flutter mobile application for structured flashcard-based learning, featuring multi-level progression, MCQ assessments, Firebase authentication, push notifications, achievement tracking, and PDF certificate generation.

---

## Features

- **6 Subject Fields** — each with 3 difficulty levels (Easy, Normal, Hard)
- **Study Cards** — 180 open-ended Q&A cards (10 per level per field), self-marked by the learner
- **Assessments & Final Exams** — 90 MCQ questions (5 per level assessment, 10 for final exam), auto-graded
- **Achievement System** — badges, milestones, and level-completion tracking
- **PDF Certificates** — printable/shareable certificates on achievement unlock
- **7 Notification Types** — daily reminders, spaced repetition, quiz results, achievements, weak topics, weekly progress, new content
- **Weekly Reports & Stats** — visual progress summaries
- **Bookmarks** — save cards for later review
- **Admin Panel** — CRUD for fields, study cards, and assessment questions (admin only)
- **Auth** — Firebase Email/Password + Google Sign-In

## Getting Started

### Prerequisites

- Flutter SDK 3.0+
- Dart SDK 3.0+
- Android Studio or VS Code with Flutter/Dart plugins
- A Firebase project with Authentication, Firestore, and Cloud Messaging enabled

### 1. Clone the repository

```bash
git clone <repository-url>
cd flashcard_app
```

### 2. Install dependencies

```bash
flutter pub get
```

### 3. Firebase setup

Install the FlutterFire CLI and configure Firebase:

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

This generates `lib/firebase_options.dart` and places `google-services.json` (Android) and `GoogleService-Info.plist` (iOS) in their respective directories.

Enable the following in your Firebase console:
- **Authentication** — Email/Password and Google sign-in providers
- **Cloud Firestore** — create a database (start in test mode for development)
- **Cloud Messaging** — enabled by default

### 4. Run the app

```bash
flutter run
```

## Admin Access

The admin account is identified by email at runtime in `lib/services/auth_service.dart`. Only the designated admin email gains CRUD access to:

- Subject fields
- Study card questions and answers
- Assessment MCQ questions and options

All other authenticated users receive learner-only access.

## Architecture

| Concern | Approach |
|---------|----------|
| State management | `provider` (ChangeNotifier) |
| Data layer | Static Dart constants (no external API required) |
| Authentication | Firebase Auth (Email/Password + Google Sign-In) |
| Push notifications | Firebase Cloud Messaging + `flutter_local_notifications` |
| Local storage | `shared_preferences` + `hive` |
| PDF generation | `pdf` + `printing` packages |

---

## Notification Types

| Type | Trigger |
|------|---------|
| Daily Reminder | App launch / scheduled daily |
| Spaced Repetition | Time elapsed since last review |
| Quiz Result | After every assessment or exam |
| Achievement | Level completion, streak milestones |
| Weak Topic | Assessment score below 60% |
| Progress Summary | Weekly summary |
| New Content | Field selection, admin content update |

Notifications persist in the notification panel and are never auto-dismissed.

## Study Cards vs Assessments

| | Study Cards | Assessments / Final Exam |
|-|-------------|--------------------------|
| Format | Open-ended Q&A | Multiple choice (4 options) |
| Count | 10 per level | 5 per level, 10 for final |
| Evaluation | Learner self-marks | System auto-grades |
| Purpose | Conceptual review | Knowledge testing |
